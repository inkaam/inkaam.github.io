Mikä on Ticorporate?

Ticorporate on JAMKin opintojakso, jossa opiskelijat toteuttavat ulkopuolisia toimeksiantoja tai kehittävät omia sovelluksia ja pelejä. Oppimisympäristö jäljittelee työelämää, ja käytetyt teknologiat vastaavat alan käytäntöjä.

Ennen Ticorporatea

Taitotasoni

Ennen Ticorporate-projektia tekninen taitotasoni oli kohtalainen. Osasin Frontendin perusteet ja olin yhdistänyt Frontendin ja Backendin toimivaksi kokonaisuudeksi Backend 1 -kurssin lopputyössä, jossa toteutin sisäänkirjautumisen, rekisteröitymisen sekä ajanvarausjärjestelmän.

Tavoitteeni

Asetin itselleni tavoitteeksi kehittyä koodaajana ja ymmärtää paremmin Frontend-kehittäjän roolia sekä sen vastuita suuremmassa projektissa.

Projekti: STUNITE

STUNITE on viiden tietojenkäsittelyopiskelijan Ticorporate-projekti syksyltä 2024. Sovellus on suunnattu tapahtumakävijöille ja -järjestäjille, tavoitteena tuoda opiskelijatapahtumat laajemmin esille ja edistää eri järjestöjen välistä yhteistyötä.

Projektissa roolit jakautuivat Backend- ja Frontend-kehittäjään (sivuroolina molemmilla testaus), pilvipalveluosaajaan, UI/UX-suunnittelija & sisällöntuottajaan (sivuroolina molemmilla Scrum Master) sekä liiketoimintaan keskittyvään osaajaan (sivuroolina Product Owner). Oma pääroolini oli Frontend-kehittäjä, ja sivuroolissani toimin siis testaajana.

Roolini Frontend koodarina

Frontend koodaajan rooli on tärkeä, koska ilman sitä käyttäjällä ei ole mitää nähtävää tai koettavaa projektista. Roolissa pääsin työskentelemään tiiviisti niin Backend osaajan kanssa, että UI/UX suunnittelijan kanssa. Näin saimme yhdessä aikaan suunnitellun näköisen ja backendin tietoja sekä metodeja käyttävän kokonaisuuden.

Oma rooli kuitenkin tuntui välillä hieman raskaalta, koska olin lähinnä yksin vastuussa Frontendissä, kun kaikilla ei ollut kokemusta asiasta. Autoimme kuitenkin toisiamme parhaamme mukaan toisen tekniikkaan keskittyvän osaajan kanssa. Sain myös myöhemmin apua sivuston CSS:ään ja sisältöön UI/UX suunnittelialta / sisällöntuottajalta.

Roolini testaajana

Toimin ryhmän toisena testaajana ja tein Frontendille yksikkötestejä sekä End-To-End testausta Cypressillä. Valitettavasti testaaminen jäi itsellä projektin loppuvaiheeseen, koska Frontend vei niin paljon aikaa.

Mitä sain aikaan?:

  • Suurimman osan sivujen HTML ja CSS
  • Normikäyttäjän sekä järjestäjän sisään- ja uloskirjautuminen
  • Normikäyttäjän rekisteröityminen
  • Normikäyttäjän profiilisivu, jossa voi muokata tietoja
  • Etusivulle kolmannen osapuolen NPM paketin avulla kalenteri ja siihe näkymään tapahtumat
  • Järjestäjäkalenteri(sekä tapahtumat listamuodossa), jossa näkyy kaikkien järjestävien alustavat ja vahvistetut tapahtumasuunnitelmat
  • Tapahtuman luominen, muokkaaminen ja poistaminen mahdollisuus tapahtuman järjestsäjille
  • Tapahtumat -sivu, jossa tulevaisuuden julkaistut ja vahvistetut tapahtumat
  • Tapahtuma-arkisto -sivu, jossa menneet julkaistut ja vahvistetut tapahtumat
  • Haku ja filteröinti Tapahtumat ja Tapahtuma-arkisto sivuille
  • Järjestäjäkalenterin filteröinti
  • Tapahtumakohtaiset sivut jokaiselle tapahtumalle, jossa löytyy tapahtumat tiedot, linkki lipunmyyntii sekä osoite kartalla
  • Sivuston responsiivisuus
  • Pieni määrä E2E ja yksikkötestejä

Esimerkkejä:

Tapahtumahaku suodatus:

UpdateFilteredEvents(eventsData: Event[]) päivittää suodatetut tapahtumat käyttäjän hakutermien ja valintojen perusteella. Se myös lajittelee tulokset aikajärjestykseen alkamispäivän mukaan. Suodatetut tapahtumat tallennetaan filteredEventData - muuttujaan. 


            updateFilteredEvents(eventsData: Event[]) {
              this.filteredEventData = computed(() => {
                const search = this.searchTerm().toLowerCase();
                const city = this.selectedCity();
                const tag = this.selectedTag();
                const { start, end } = this.selectedDateRange();
          
                let filteredEvents = eventsData.filter((event) => {
                  const matchesSearch = event.eventName.toLowerCase().includes(search);
                  const matchesCity = city ? event.city === city : true;
                  const matchesTag = tag ? event.eventTags?.includes(tag) : true;
          
                  let matchesDate = true;
                  if (start || end) {
                    const eventStartDate = this.parseCustomDate(event.date);
                    matchesDate =
                      (!start || eventStartDate >= new Date(start)) &&
                      (!end || eventStartDate <= new Date(end));
                  }
          
                  const currentDate = new Date();
                  currentDate.setHours(0, 0, 0, 0);
          
                  // Use endingDate if available, otherwise fallback to event.date (starting date)
                  const eventEndDate = event.endingDate
                    ? this.parseCustomDate(event.endingDate)
                    : this.parseCustomDate(event.date); // Fall back to starting date
                  eventEndDate.setHours(0, 0, 0, 0);
          
                  const isFutureEvent = eventEndDate >= currentDate;
          
                  return (
                    matchesSearch &&
                    matchesCity &&
                    matchesTag &&
                    matchesDate &&
                    isFutureEvent
                  );
                });
          
                filteredEvents = filteredEvents.sort((a, b) => {
                  const dateA = this.parseCustomDate(a.date);
                  const dateB = this.parseCustomDate(b.date);
                  return dateA.getTime() - dateB.getTime();
                });
          
                return filteredEvents;
              });
            }

Järjestäjäkalenteri:

eventsUpdated() hakee kaikki tapahtumat ja suodattaa ne käyttäjän valintojen perusteella. Se suodattaa tapahtumat valitun kaupungin (selectedCity), valittujen tunnisteiden (selectedTags) ja valitun järjestäjän (selectedOrganizer) perusteella. Tämän jälkeen funktio jakaa tapahtumat tuleviin ja menneisiin tapahtumiin: tulevat tapahtumat järjestetään aikajärjestykseen vanhimmasta uusimpaan, ja menneet tapahtumat järjestetään uusimmasta vanhimpaan. Lopuksi se päivittää kalenterin tapahtumatiedot suodatettujen tapahtumien perusteella. Funktio kutsutaan aina, kun käyttäjä muuttaa kaupunkia, tunnisteita tai järjestäjää (onCityChange, onTagChange, onOrganizerChange). 


eventsUpdated() {
    const eventsData = this.events();
    const currentDate = new Date();

    let filteredEvents = eventsData;

    if (this.selectedCity) {
      filteredEvents = filteredEvents.filter(
        (event) => event.city.toLowerCase() === this.selectedCity.toLowerCase()
      );
    }

    if (this.selectedTags && this.selectedTags.length > 0) {
      filteredEvents = filteredEvents.filter((event) =>
        event.eventTags.includes(this.selectedTags)
      );
    }

    this.noEventsForTag = filteredEvents.length === 0;

    if (this.selectedOrganizer) {
      filteredEvents = filteredEvents.filter((event) => {
        return (
          event.organizationName.toLowerCase() ===
          this.selectedOrganizer.toLowerCase()
        );
      });
    }

    this.upcomingEvents = filteredEvents
      .filter((event) => {
        const eventDate = new Date(this.formatDateToISO(event.date));
        return eventDate >= currentDate;
      })
      .sort((a, b) => {
        const dateA = new Date(
          this.formatDateToISO(a.date) + 'T' + a.startingTime
        );
        const dateB = new Date(
          this.formatDateToISO(b.date) + 'T' + b.startingTime
        );
        return dateA.getTime() - dateB.getTime();
      });

    this.pastEvents = filteredEvents
      .filter((event) => {
        const eventDate = new Date(this.formatDateToISO(event.date));
        return eventDate < currentDate;
      })
      .sort((a, b) => {
        const dateA = new Date(
          this.formatDateToISO(a.date) + 'T' + a.startingTime
        );
        const dateB = new Date(
          this.formatDateToISO(b.date) + 'T' + b.startingTime
        );
        return dateB.getTime() - dateA.getTime();
      });

    if (filteredEvents.length > 0) {
      this.calendarOptions.events = filteredEvents.map((event) => ({
        title: event.eventName,
        date: this.formatDateToISO(event.date),
        color: event.status === 'Varattu' ? '#fe7775' : '#f0d37c',
        url: `/events/${event._id}`,
        extendedProps: {
          startingTime: event.startingTime,
          venue: event.venue,
          date: event.date,
          organizationName: event.organizationName,
          eventId: event._id,
        },
      }));
    } else {
      this.calendarOptions.events = [];
    }
  }
          
  onCityChange(city: string): void {
    this.selectedCity = city;
    this.eventsUpdated();
  }
  onTagChange(selectedTag: string): void {
    this.selectedTags = selectedTag;
    this.eventsUpdated();
  }
  onOrganizerChange(selectedOrganizer: string): void {
    this.selectedOrganizer = selectedOrganizer;
    this.eventsUpdated();
  }

Tapahtuman Alustava/Vahvistettu statuksen vaikutus:

getPublishedEvents() hakee tapahtumat tietokannasta, mutta suodattaa ne niin, että vain vahvistetut ja julkaistut tapahtumat jäävät mukaan. Tätä kutsua käytetään Tapahtumat -sivulla, jotta saadaan vain julkaistut tapahtumat.

Järjestäjäkalenterissa on hyödynnetty värejä selkeyttämään onko tapahtumalla Vahvistettu vai Alustava suunnitelma status. 


            getPublishedEvents(): Observable {
              return this.http.get(this.apiUrl).pipe(
                map((events) =>
                  events.filter((event) => {
                    const now = new Date();
                    const publishDate = new Date(event.publishDateTime);
          
                    return (
                      event.status === 'Varattu' &&
                      event.publishDateTime != null &&
                      publishDate <= now
                    );
                  })
                )
              );
            }

Mobiili

Mitä opin

  • Työskentelyä ryhmässä, jossa jokaisella on eri ns. eri homma, eikä tehdä paljoa samaa.
  • Vähän parempi ymmärrys Yksikkötesteistä ja miten niitä voi tehdä Angularissa.
  • Kolmansien osapuolien NPM pakettien hyödyntäminen isomassa projektissa. (Niiden etsiminen, testaaminen, käyttöönotta ja muokkaaminen juuri projektiin sopivaksi)
  • Tajuan vähän paremmin nyt Angular signaaleja ja niiden käyttöä.

Oivalluksia

  • Kannattaa käyttää aikaa suunnitteluun.
  • Koodia kannattaa kommentoida hyvin alusta asti, mikäli aikaa on.
  • Koodi saattaa toimia localhostilla, mutta AWS päällä ei.
  • Jos mahdollista, työ kannattaa deployata AWS päälle mahdollisimman pian kehitysvaiheessa, jotta yllä oleva ongelma ei ilmenisi liian myöhään.

Tavoitteiden toteutuminen

Vaikka asetin itselle aika ympäripyöreitä tavoitteita, koen että olen kehittynyt paljon Frontend koodaajana ja myös Angular on tullut tutummaksi. Tavoittelin parempaa ymmärrystä Frontend koodarin hommista ja vastuista, ja se kyllä tuli tavoitettua. Suurin osa Frontendin toiminnallisuuksista ja HTML rakenteista on tekemiäni sekä tein myös paljon CSS tyylittelyjä. Frontend oli kuitenkin pääasiassa minun vastuulla ja opin kuinka paljon hommaa siinä voi olla isommassa projektissa yhdelle ihmiselle.

Ennen Ticorporatea pisimmät/isoimmat koodaus projektini ovat olleet Frontend sovelluskehityksen ja Backend 1 opintojaksojen lopputyöt, joten Ticorporaten työmäärä tuntui mahdottoman suurelta. Siitä huolimatta olen sitä mieltä, että tein reippaasti hommia ja sain yllättävän paljon aikaan. Koen olevani paljon sujuvampi sekä itsevarmempi koodaaja kuin ennen Ticorporatea.

Projektin jälkeen jäi kiinnostamaan millaista olisi työskennellä alusta alkaen Frontendiä jonkun kanssa samaan aikaan, lähinnä millaista työnjakoa silloin kannattaa tehdä ja millaisia käytänteitä tarvitaan.

Omat vahvuudet

Vahvuuksiani ovat Frontend-kehityksessä erityisesti toiminnallisuuksien luominen sekä HTML-rakenteen ja CSS-tyylittelyn hallinta. Tällä hetkellä osaan Angularia, mutta uskon pystyväni oppimaan nopeasti uusia Frontend-frameworkkejä. Olen myös vahva tiimityöskentelijä ja nautin muiden auttamisesta. Rakastan uuden oppimista ja koodiongelmien ratkomista, vaikka ne välillä tuottavatkin haasteita ja harmaita hiuksia.
